===== Phase 5G contract lifecycle rehearsal =====
Statuses (model): DRAFT, PENDING, IN_REVIEW, APPROVED, ACTIVE, EXPIRED, TERMINATED, COMPLETED, CANCELLED
Transition graph (implementation == documentation, contract_lifecycle.CONTRACT_STATUS_TRANSITIONS):
  DRAFT     -> IN_REVIEW, PENDING, CANCELLED
  PENDING   -> IN_REVIEW, APPROVED, ACTIVE, CANCELLED
  IN_REVIEW -> APPROVED, PENDING, ACTIVE, DRAFT, CANCELLED
  APPROVED  -> ACTIVE, CANCELLED
  ACTIVE    -> EXPIRED, TERMINATED, COMPLETED
  EXPIRED/TERMINATED/COMPLETED/CANCELLED -> (terminal; none)
  DRAFT->ACTIVE rejected; terminal reopen rejected; same-status idempotent no-op.

Approval applicability rule: ACTIVE requires an APPROVED ApprovalRequest for THIS contract.
  Blocked: none / pending / rejected / approval-of-other-contract / approval-in-other-org.

Signature applicability rule (FIXED this section): evaluate the CURRENT request per signer
  (latest by created_at; no packet model exists). Per signer's current request:
    SIGNED -> satisfied ; CANCELLED/EXPIRED -> withdrawn (ignored) ;
    PENDING/SENT/VIEWED -> in-flight (BLOCKS) ; DECLINED -> active refusal (BLOCKS).
  Verified: no-sig activates; one-signed activates; one-pending blocks;
  signed+old-cancelled activates; signed+expired-historical activates;
  superseded(declined->re-signed) activates; current-declined blocks; abandoned-pending blocks;
  multi-signer partly-signed blocks; all-signed activates; all-cancelled(withdrawn) activates;
  other-contract in-flight does NOT block.

Authorization: unauthenticated rejected; cross-tenant actor rejected (no status change).
Audit: exactly ONE chained, tenant-attributed contract.status_changed per success; none on rejection.
Stale-instance: service re-reads/locks DB (select_for_update of=self) -> uses DB state not stale
  in-memory; approval added after instance load is seen; DB status change after load is honored.
Bypass paths: HTML contract_update (illegal blocked), bulk API (illegal -> 400; valid -> service+audit).

Concurrency (PostgreSQL, real threads/connections):
  duplicate activation -> commits once (1 audit, 0 errors)
  racing EXPIRED vs COMPLETED from ACTIVE -> one winner (1 audit), loser InvalidContractTransition (no DB error)
